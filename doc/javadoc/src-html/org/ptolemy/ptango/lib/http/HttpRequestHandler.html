078/** An actor that handles an HTTP request by producing output
079 *  and waiting for an input that provides a response.
080 *  This actor requires that the model that contains
081 *  it include an instance of {@link WebServer}, which discovers this actor and
082 *  delegates HTTP requests to a servlet that this actor creates.
083 *  It also requires that the model containing this actor provide
084 *  exactly one input to at least one of the input ports in response to
085 *  every output request,
086 *  and that such responses are delivered to its input in the very
087 *  next microstep, or at least before any other request is made
088 *  by the WebServer.
089 *  <p>
090 *  The <i>path</i> parameter specifies which HTTP requests will
091 *  be delegated to this actor. If the base URL for the web server
092 *  is "http://localhost:8080", say, then request of the form
093 *  "http://localhost:8080/<i>path</i>" will be delegated to this
094 *  actor. It is an error if two instances of this actor to have the
095 *  same <i>path</i> in a model?
096 *  <p>
097 *  When this actor receives an HTTP request from the WebServer, it
098 *  issues a request for the director to fire it at the greater of
099 *  the current time of the director or the time elapsed since
100 *  the last invocation of initialize() (in seconds).
101 *  When the actor fires, it produces on its output ports the details
102 *  of the request, time stamped by the elapsed time since the model
103 *  started executing. It expects to be in a model that will send
104 *  it data to input ports with the response to HTTP request.
105 *  If that response does not arrive within <i>timeout</i>
106 *  (default 30000) milliseconds, then this actor will a issue
107 *  timeout response and will discard the response when it eventually
108 *  arrives.
109 *  <p>
110 *  This actor is designed to be used in a DE model (with a DEDirector).
111 *  Since a model using this actor must deliver a response for
112 *  every request, the model must put this actor in a feedback
113 *  loop. The requests appear on the outputs of this actor,
114 *  and the responses appear on the inputs. Each such feedback
115 *  loop is required to include an instance of {@link MicrostepDelay}
116 *  in order to break the causality loop that the feedback loop would
117 *  otherwise incur.
118 *  The downstream model should be used to construct a response.
119 *  For example, to simply serve a web page, put a
120 *  {@link FileReader} actor to read the response from a local
121 *  file and a {@link MicrostepDelay} downstream in a feedback
122 *  loop connected back to the response input.
123 *  <p>
124 *  Some of the output ports ({@link #parameters} and {@link #headers})
125 *  produce records and constrain their output type to be less than
126 *  or equal to an empty record. This does not, however, provide
127 *  enough information for type inference. If the downstream model
128 *  that receives these records <i>requires</i> particular fields
129 *  in the record, then putting a {@link RecordDisassembler} actor downstream
130 *  to extract the field, and enabling backward type inference at the top
131 *  level of the model will result in a constraint on this port that the
132 *  record contain the specified field.  A malformed URL that does
133 *  not contain the specified record fields will result in a response
134 *  400 Bad Request, meaning "the request could not be understood
135 *  by the server due to malformed syntax.
136 *  The client SHOULD NOT repeat the request without modifications."
137 *  If the downstream model does not require
138 *  any particular fields, but will rather examine any fields that
139 *  are provided, for example in a {@link ptolemy.actor.lib.jjs.JavaScript} actor, then
140 *  you can declare the output ports of this actor to be of type "record",
141 *  or you can declare the input port of the downstream actor to be
142 *  of type "record" and enable backward type inference. In this case,
143 *  any fields that are provided will be passed downstream as a record.
144 *
145 *  @author Elizabeth Latronico, Edward A. Lee, and Marten Lohstroh
146 *  @version $Id$
147 *  @since Ptolemy II 10.0
148 *  @Pt.ProposedRating Red (ltrnc)
149 *  @Pt.AcceptedRating Red (ltrnc)
150 *  @see org.ptolemy.ptango.lib.WebServer
151 */
152public class HttpRequestHandler extends TypedAtomicActor
153        implements HttpService, ExceptionSubscriber {

388    /** React to a change in an attribute.  In this case, check the
389     *  value of the <i>path</i> attribute to make sure it is a valid URI.
390     *  @param attribute The attribute that changed.
391     *  @exception IllegalActionException If the change is not acceptable
392     *   to this container (not thrown in this base class).
393     */
394    @Override
395    public void attributeChanged(Attribute attribute)
396            throws IllegalActionException {
397        if (attribute == path) {
398            String pathValue = ((StringToken) path.getToken()).stringValue();
399            try {
400                // Path should start with a "/", or be "*"
401                if (!pathValue.trim().equals("")) {
402                    if (!pathValue.trim().startsWith("/")) {
403                        _URIpath = URI.create("/" + pathValue);
404                    } else {
405                        _URIpath = URI.create(pathValue);
406                    }
407                } else {
408                    _URIpath = URI.create("/*");
409                }
410            } catch (IllegalArgumentException e2) {
411                throw new IllegalActionException(this,
412                        "Path is not a valid URI: " + pathValue);
413            }
414        } else {
415            super.attributeChanged(attribute);
416        }
417    }

450    /** Generate an HTTP response for the client if an exception occurs in
451     *  the model and there is a {@link CatchExceptionAttribute} in the model.
452     *  No response will be received on the input port in the event of such an
453     *  exception.
454     *  @param policy The exception handling policy of the exception handler;
455     *   see {@link CatchExceptionAttribute}
456     *  @param exception  The exception that occurred
457     *  @return True since a response is always sent to the client, and there
458     *    there are no operations that throw exceptions
459     */
460    @Override
461    public synchronized boolean exceptionOccurred(String policy,
462            Throwable exception) {
463        // If there is a pending request,
464        // For "restart" policy, generate an error page with retry
465        // For other policies, generate a Server Error error page
466        // These method calls notifyAll() so that the response will be sent
467        // by the servlet thread
468        if (_pendingRequest != null) {
469            if (policy.equals(CatchExceptionAttribute.RESTART)) {
470                _respondWithRetryMessage(10);
471            } else {
472                _respondWithServerErrorMessage();
473            }
474            // Request is no longer pending.
475            _pendingRequest = null;
476        }
477        return true;
478    }

507    /** If there are input tokens on the {@link #responseBody},
508     *  {@link #setCookies}, or {@link #responseCode} ports and there is a pending
509     *  request, then send a response. Otherwise, if the servlet has received
510     *  an HTTP request, then also produce on the output ports
511     *  the details of the request.
512     *  @exception IllegalActionException If sending the outputs fails.
513     */
514    @Override
515    public synchronized void fire() throws IllegalActionException {
516        // The methods of the servlet are invoked in another
517        // thread, so we synchronize on this actor for mutual exclusion.
518        super.fire();
519
520        boolean responseFound = false;
521        HttpResponseItems responseData = new HttpResponseItems();
522        // Check for new cookies on the setCookies port.
523        if (setCookies.getWidth() > 0 && setCookies.hasToken(0)) {
524            RecordToken cookieToken = (RecordToken) setCookies.get(0);
525            // Note that we issue a response if we get cookies only.
526            // If there is no response input, then the response will be an empty string.
527            responseFound = true;
528            responseData.hasNewCookies = true;
529            responseData.cookies = cookieToken;
530            // Default response has empty text.
531            responseData.response = "";
532            if (_debugging) {
533                _debug("Received cookies on the setCookies input port: "
534                        + cookieToken);
535            }
536        }
537
538        // Handle the response data.
539        if (responseBody.hasToken(0)) {
540            responseData.response = ((StringToken) responseBody.get(0))
541                    .stringValue();
542            responseFound = true;
543            if (_debugging) {
544                _debug("Received response on the response input port: "
545                        + responseData.response);
546            }
547        }
548        // Handle the code input.
549        if (responseCode.getWidth() > 0 && responseCode.hasToken(0)) {
550            responseData.statusCode = ((IntToken) responseCode.get(0))
551                    .intValue();
552            responseFound = true;
553            if (_debugging) {
554                _debug("Received response code: " + responseData.statusCode);
555            }
556        }
557        // Handle the responseContentType input.
558        responseContentType.update();
559        responseData.contentType = ((StringToken) responseContentType
560                .getToken()).stringValue();
561
562        // Handle the responseHeaders input.
563        responseHeaders.update();
564        responseData.headers = (RecordToken) responseHeaders.getToken();
565
566        if (responseFound) {
567            if (_pendingRequest == null) {
568                // There is no pending request, so ignore the response.
569                if (_debugging) {
570                    _debug("Discarding the response because there is no pending request.");
571                }
572            } else {
573                _response = responseData;
574                // Indicate that the request has been handled.
575                _pendingRequest = null;
576                // Notify the servlet thread.
577                notifyAll();
578            }
579        }
580
581        // If there is a new request, produce outputs for that request,
582        // including any cookies from that request.
583        if (_newRequest != null) {
584            // Indicate that this request is now pending.
585            _pendingRequest = _newRequest;
586            _newRequest = null;
587
588            // To avoid the risk of producing two outputs at the same superdense time,
589            // check the time of the last output.
590            Director director = getDirector();
591            // Do not do this if the director does not support superdense time.
592            // In that case, there is no issue with multiple outputs.
593            if (director instanceof SuperdenseTimeDirector) {
594                Time currentTime = director.getModelTime();
595                int microstep = ((SuperdenseTimeDirector) director).getIndex();
596                if (_lastOutputTime != null
597                        && _lastOutputTime.equals(currentTime)
598                        && microstep == _lastMicrostep) {
599                    director.fireAtCurrentTime(this);
600                    // Leave the request queue alone and don't produce outputs now.
601                    return;
602                }
603                _lastMicrostep = microstep;
604                _lastOutputTime = currentTime;
605            }
606
607            if (_debugging) {
608                _debug("Sending request method: " + _pendingRequest.method);
609                _debug("Sending request URI: " + _pendingRequest.requestURI);
610            }
611            // Produce header output.
612            if (_pendingRequest.headers != null
613                    && _pendingRequest.headers.length() > 0) {
614                if (!headers.sinkPortList().isEmpty()) {
615                    if (_debugging) {
616                        _debug("Sending request header: "
617                                + _pendingRequest.headers);
618                    }
619                    try {
620                        headers.send(0, _pendingRequest.headers);
621                    } catch (TypedIOPort.RunTimeTypeCheckException ex) {
622                        // Parameters provided do not match the required type.
623                        // Construct an appropriate response.
624                        _respondWithBadRequestMessage(_pendingRequest.headers,
625                                headers.getType(), "header");
626                        return;
627                    }
628                }
629            }
630            // Produce requestor output.
631            if (_pendingRequest.requestor != null
632                    && _pendingRequest.requestor.length() > 0) {
633                if (_debugging) {
634                    _debug("Sending requestor identification: "
635                            + _pendingRequest.requestor);
636                }
637                requestor.send(0, new StringToken(_pendingRequest.requestor));
638            }
639            if (!parameters.sinkPortList().isEmpty()) {
640                if (_debugging) {
641                    _debug("Sending request parameters: "
642                            + _pendingRequest.parameters);
643                }
644                // Send parameters, but handle type errors locally.
645                try {
646                    parameters.send(0, _pendingRequest.parameters);
647                } catch (TypedIOPort.RunTimeTypeCheckException ex) {
648                    // Parameters provided do not match the required type.
649                    // Construct an appropriate response.
650                    _respondWithBadRequestMessage(_pendingRequest.parameters,
651                            parameters.getType(), "parameters");
652                    return;
653                }
654            }
655            if (_pendingRequest.cookies != null
656                    && _pendingRequest.cookies.length() > 0) {
657                if (!cookies.sinkPortList().isEmpty()) {
658                    if (_debugging) {
659                        _debug("Sending request cookies: "
660                                + _pendingRequest.cookies);
661                    }
662                    try {
663                        cookies.send(0, _pendingRequest.cookies);
664                    } catch (TypedIOPort.RunTimeTypeCheckException ex) {
665                        // Parameters provided do not match the required type.
666                        // Construct an appropriate response.
667                        _respondWithBadRequestMessage(_pendingRequest.cookies,
668                                cookies.getType(), "cookies");
669                        return;
670                    }
671                }
672            }
673            // Send the body of the request.
674            body.send(0, new StringToken(_pendingRequest.body));
675            // Send the following only if the above succeeded.
676            // Otherwise, we will send two responses to this one request.
677            method.send(0, new StringToken(_pendingRequest.method));
678            uri.send(0, new StringToken(_pendingRequest.requestURI));
679        }
680        if (_debugging) {
681            _debug("Ending fire.");
682        }
683    }

730    /** Handle an HTTP request by recording the request data, requesting
731     *  a firing of this actor at the current time, waiting for a response
732     *  to be created, and then sending the response. Normally, the actor
733     *  will fire once to produce the request data on its output ports,
734     *  then fire again to receive the response data on its input ports.
735     *  Upon that second firing, the thread in which this method is invoked
736     *  will be notified, and this method will retrieve the response data
737     *  that were provided at the input ports and send the response to the
738     *  requestor.
739     *  See <a href="http://docs.oracle.com/javaee/6/api/javax/servlet/http/HttpServletRequest.html">http://docs.oracle.com/javaee/6/api/javax/servlet/http/HttpServletRequest.html</a>.
740     *  @param request The HTTP request.
741     *  @param response The HTTP response to write to.
742     *  @param type The type of request. 0 for get, 1 for post.
743     *  @exception ServletException  If there is a problem reading from the
744     *  servlet request or other servlet problem
745     *  @exception IOException  If there is a problem writing to the servlet
746     *  response
747     */
748    protected void _handleRequest(HttpServletRequest request,
749            HttpServletResponse response, String type)
750            throws ServletException, IOException {
751        // The following code block is synchronized on the enclosing
752        // actor. This lock _is_ released while waiting for the response,
753        // allowing the fire method to execute its own synchronized blocks.
754        synchronized (HttpRequestHandler.this) {
755            // FIXME: What if there is already a _newRequest that has not been handled?
756            // Is this possible?
757            // This will simply overwrite it. For now, print a warning to standard error.
758            if (_newRequest != null) {
759                System.err.println(getFullName()
760                        + ": WARNING. Discarding HTTP request that has not yet been handled.");
761            }
762            _newRequest = new HttpRequestItems();
763
764            _newRequest.requestURI = request.getRequestURI();
765            _newRequest.method = type;
766
767            if (_debugging) {
768                _debug("**** Handling a " + type + " request to URI "
769                        + _newRequest.requestURI);
770            }
771
772            try {
773                // Read cookies from the request and store.
774                _newRequest.cookies = _readCookies(request);
775
776                // Read header information from the request and store.
777                _newRequest.headers = _readHeaders(request);
778
779                // Get the name or IP address of the requestor.
780                _newRequest.requestor = request.getRemoteHost();
781
782                // Get the parameters that have been either posted or included
783                // as assignments in a get using the URL syntax ...?name=value.
784                // Note that each parameter name may have more than one value,
785                // hence the array of strings.
786                _newRequest.parameters = _readParameters(request);
787
788                // Read the body of the request. This may be an empty string.
789                _newRequest.body = _readBody(request);
790
791                // Figure out what time to request a firing for.
792                long elapsedRealTime = System.currentTimeMillis()
793                        - _initializeRealTime;
794
795                // Assume model time is in seconds, not milliseconds.
796                Time timeOfRequest = _initializeModelTime
797                        .add(elapsedRealTime / 1000.0);
798
799                if (_debugging) {
800                    _debug("**** Request firing at time " + timeOfRequest);
801                }
802
803                // Request a firing of this actor (to produce output data).
804                // Note that fireAt() will modify the requested firing time
805                // if it is in the past.
806                // Note that past firing times might not be modified
807                // if ThreadedComposite actors are used (since the request
808                // might be at a present time inside the ThreadedComposite,
809                // but a past time for the top-level model).
810                getDirector().fireAt(HttpRequestHandler.this, timeOfRequest);
811            } catch (IllegalActionException e) {
812                _newRequest = null;
813                response.sendError(HttpServletResponse.SC_BAD_REQUEST,
814                        e.getMessage());
815                return;
816            }
817            //////////////////////////////////////////////////////
818            // Wait for a response.
819            // We are assuming every request gets exactly one response.
820            // Normally, the actor will have fired twice now, first in
821            // response to our firing request, and then again in response
822            // to input data. The first firing will have sent the
823            // _newRequest data to the output ports, moved the
824            // data to _pendingRequest, and set _newRequest to null.
825            // The second firing could conceivable have another _newRequest,
826            // if this method is called again during the wait() call below,
827            // in which case the second firing will have input data for
828            // a response to the request in _pendingRequest, and new
829            // request data in _newRequest, which it will send to its
830            // output ports.
831            while (_response == null) {
832                if (_debugging) {
833                    _debug("**** Waiting for a response");
834                }
835                try {
836                    // Timeout after time given by timeout parameter.
837                    // Unfortunately, we can't tell whether the timeout
838                    // occurred unless we record the current time.
839                    long startTime = System.currentTimeMillis();
840                    long timeoutValue = 30000L;
841                    try {
842                        timeoutValue = ((LongToken) timeout.getToken())
843                                .longValue();
844                    } catch (IllegalActionException e) {
845                        // Ignore and use default of 30 seconds.
846                    }
847                    // Wait for notification that a _response has been
848                    // constructed.
849                    HttpRequestHandler.this.wait(timeoutValue);
850                    if (_response == null && System.currentTimeMillis()
851                            - startTime >= timeoutValue) {
852                        // A timeout has occurred, and there is still no _reponse.
853                        // This means that the second firing never occurred, so no
854                        // response data have been provided.
855                        if (_debugging) {
856                            _debug("**** Request timed out.");
857                        }
858
859                        // Set the content length (enables persistent
860                        // connections) and send the buffer
861                        response.sendError(
862                                HttpServletResponse.SC_REQUEST_TIMEOUT,
863                                "Request Timeout (408)");
864
865                        // Indicate that there is no longer a pending request.
866                        // This ensures that if the second firing later occurs,
867                        // its response data are not sent.
868                        _pendingRequest = null;
869
870                        return;
871                    }
872                } catch (InterruptedException e) {
873                    if (_debugging) {
874                        _debug("*** Request thread interrupted.");
875                    }
876                    response.sendError(
877                            HttpServletResponse.SC_INTERNAL_SERVER_ERROR,
878                            "Get request thread interrupted (Internal Server Error, 500)");
879
880                    // Indicate that there is no longer a pending request or response.
881                    _pendingRequest = null;
882                    _response = null;
883
884                    return;
885                }
886            }
887
888            // At this point, we are assured that _response is not null.
889            response.setStatus(_response.statusCode);
890            response.setContentType(_response.contentType);
891
892            // Write all the headers to the response.
893            if (_response.headers != null) {
894                for (String name : _response.headers.labelSet()) {
895                    Token token = _response.headers.get(name);
896                    if (token instanceof StringToken) {
897                        response.addHeader(name,
898                                ((StringToken) token).stringValue());
899                    } else if (token != null) {
900                        response.addHeader(name, token.toString());
901                    }
902                }
903            }
904
905            // Write all cookies to the response, if there are some new
906            // cookies to write
907            if (_response.hasNewCookies) {
908                _writeCookies(_response.cookies, response);
909            }
910            if (_debugging) {
911                _debug("**** Servet received response: " + _response.response);
912            }
913
914            // Set up a buffer for the output so we can set the length of
915            // the response, thereby enabling persistent connections
916            // http://docstore.mik.ua/orelly/java-ent/servlet/ch05_03.htm
917            ByteArrayOutputStream bytes = new ByteArrayOutputStream();
918            PrintWriter writer = new PrintWriter(bytes, true); // true forces flushing.
919
920            // Set the content length (enables persistent
921            // connections) and send the buffer
922            // Use print rather than println to prevent an extra \n on the response.
923            writer.print(_response.response);
924            // Printwriter only flushes println() automatically. So force a flush.
925            writer.flush();
926            response.setContentLength(bytes.size());
927            bytes.writeTo(response.getOutputStream());
928
929            // Close the PrintWriter
930            // Close the ByteArrayOutputStream to avoid a
931            // warning, though this is not necessary since
932            // ByteArrayOutputStream's close() method has no
933            // effect
934            // http://docs.oracle.com/javase/6/docs/api/java/io/ByteArrayOutputStream.html#close%28%29
935            writer.close();
936            bytes.close();
937
938            // Indicate response has been handled.
939            _response = null;
940        }
941    }

962    /** Read the Cookies from the HttpServletRequest, construct
963     *  a record token with one field for each name in the requestedCookies
964     *  parameter and the value given by the request, or if there is no
965     *  value, with an empty string as a value.
966     *  @param request  The HttpServletRequest to read Cookies from.  The
967     *   HttpServletRequest can be of any type - i.e. both GET and POST
968     *   requests are allowed to have Cookies.
969     *  @return A record of cookies.
970     *  @exception IllegalActionException If construction of the record token fails.
971     */
972    protected RecordToken _readCookies(HttpServletRequest request)
973            throws IllegalActionException {
974        ArrayToken labels = (ArrayToken) requestedCookies.getToken();
975        if (labels.length() == 0) {
976            // No cookies requested.
977            // Return an empty record.
978            return RecordToken.EMPTY_RECORD;
979        }
980        // First, provide default empty string values for requested cookies.
981        LinkedHashMap<String, Token> map = new LinkedHashMap<String, Token>();
982        StringToken emptyString = new StringToken("");
983        for (int i = 0; i < labels.length(); i++) {
984            String label = ((StringToken) labels.getElement(i)).stringValue();
985            map.put(label, emptyString);
986        }
987        // Next, override these default values with any cookies provided in the request.
988        for (Cookie cookie : request.getCookies()) {
989            // Cookie must have a name
990            if (cookie.getName() != null && !cookie.getName().isEmpty()) {
991                String label = cookie.getName();
992                if (map.containsKey(label)) {
993                    // Override the default value.
994                    map.put(label, new StringToken(cookie.getValue()));
995                }
996            }
997        }
998        return new RecordToken(map);
999    }

1001    /** Read the header information from the HttpServletRequest, construct
1002     *  a record token with one field for each header name. If a header name
1003     *  has just one value, the field value will be a string. Otherwise,
1004     *  it will be an array of strings.
1005     *  @param request  The HttpServletRequest to read header information from.
1006     *  @return A record of header fields.
1007     *  @exception IllegalActionException If construction of the record token fails.
1008     */
1009    protected RecordToken _readHeaders(HttpServletRequest request)
1010            throws IllegalActionException {
1011        LinkedHashMap<String, Token> map = new LinkedHashMap<String, Token>();
1012        Enumeration<String> names = request.getHeaderNames();
1013        while (names.hasMoreElements()) {
1014            String name = names.nextElement();
1015            LinkedList<StringToken> valueList = new LinkedList<StringToken>();
1016            Enumeration<String> values = request.getHeaders(name);
1017            while (values.hasMoreElements()) {
1018                valueList.add(new StringToken(values.nextElement()));
1019            }
1020            if (valueList.isEmpty()) {
1021                // Do not include empty headers.
1022                continue;
1023            }
1024            if (valueList.size() == 1) {
1025                map.put(name, valueList.get(0));
1026            } else {
1027                map.put(name, new ArrayToken(
1028                        valueList.toArray(new StringToken[valueList.size()])));
1029            }
1030        }
1031        if (map.isEmpty()) {
1032            // Return an empty record.
1033            return RecordToken.EMPTY_RECORD;
1034        }
1035        return new RecordToken(map);
1036    }

1038    /** Read the parameters from the HttpServletRequest, construct
1039     *  a record token containing the parameters, and return that record.
1040     *  @param request  The HttpServletRequest to read parameters from.  The
1041     *   HttpServletRequest can be of any type - i.e. both GET and POST
1042     *   requests are allowed to have parameters.
1043     *  @return A record of parameters.
1044     *  @exception IllegalActionException If construction of the record token fails.
1045     */
1046    protected RecordToken _readParameters(HttpServletRequest request)
1047            throws IllegalActionException {
1048        // FIXME:  This currently treats all values as StringTokens,
1049        // which they may not be.  How to either 1) correctly determine
1050        // the type of the token, or 2) create a token of unspecified
1051        // type but still be able to set its expression?  Class
1052        // Token does not have the setExpression() method
1053        Map<String, String[]> parameterMap = request.getParameterMap();
1054        // Convert these parameters to a record token to send to
1055        // the appropriate output.
1056        Set<String> names = parameterMap.keySet();
1057        String[] fieldNames = new String[names.size()];
1058        Token[] fieldValues = new Token[names.size()];
1059        int i = 0;
1060        for (String name : names) {
1061            fieldNames[i] = name;
1062            String[] values = parameterMap.get(name);
1063            if (values == null || values.length == 0) {
1064                fieldValues[i] = StringToken.NIL;
1065            } else if (values.length == 1) {
1066                fieldValues[i] = new StringToken(values[0]);
1067            } else {
1068                // More than one value. Use an array.
1069                StringToken[] arrayEntries = new StringToken[values.length];
1070                for (int j = 0; j < values.length; j++) {
1071                    arrayEntries[j] = new StringToken(values[j]);
1072                }
1073                ArrayToken array = new ArrayToken(arrayEntries);
1074                fieldValues[i] = array;
1075            }
1076            i++;
1077        }
1078        return new RecordToken(fieldNames, fieldValues);
1079    }

1082     *  If the value of a cookie is an empty string, then clear the cookie instead.
1083     *  @param cookies The cookies.
1084     *  @param response The HttpServletResponse to write the cookies to.
1085     */
1086    protected void _writeCookies(RecordToken cookies,
1087            HttpServletResponse response) {
1088        // TODO:  Allow permanent cookies.  Current implementation produces
1089        // session cookies.  Session cookies are stored in
1090        // the browser's memory, and are erased when the browser is closed
1091        // http://www.javascriptkit.com/javatutors/cookie.shtml
1092        for (String label : cookies.labelSet()) {
1093            String value;
1094            Token recordValue = cookies.get(label);
1095            if (recordValue instanceof StringToken) {
1096                value = ((StringToken) recordValue).stringValue();
1097            } else {
1098                value = recordValue.toString();
1099            }
1100            // Clear the cookie if the value is empty
1101            // TODO:  Should there be an explicit clear cookie port?
1102            // Is there a scenario where we want the cookie to exist,
1103            // but to have an empty string for the value?
1104            if (value.isEmpty()) {
1105                Cookie cookie = new Cookie(label, "");
1106                cookie.setMaxAge(0);
1107                cookie.setPath("/"); // Cookie visible to all web pages
1108                response.addCookie(cookie);
1109            } else {
1110                // FIXME:  Special character handling?  Copied from previous
1111                // code. Should cookies be encoded using URLEncoder.encode()?
1112                if (value.indexOf("\"") != -1) {
1113                    value = value.replace("\"", "");
1114                }
1115                Cookie cookie = new Cookie(label, value);
1116                cookie.setPath("/");
1117                response.addCookie(cookie);
1118            }
1119        }

1145    /** Issue a response to the current request indicating malformed syntax.
1146     *  According to: {@link www.w3.org/Protocols/rfc2616/rfc2616-sec10.html},
1147     *  the correct response is
1148     *  10.4.1 400 Bad Request, "The request could not be understood
1149     *  by the server due to malformed syntax.
1150     *  The client SHOULD NOT repeat the request without modifications."
1151     *  @param record The record that was received.
1152     *  @param expectedType The type of record expected.
1153     *  @param what What triggered the error ("parameters" or "cookies").
1154     */
1155    private void _respondWithBadRequestMessage(RecordToken record,
1156            Type expectedType, String what) {
1157        _response = new HttpResponseItems();
1158        _response.statusCode = HttpServletResponse.SC_BAD_REQUEST;
1159        StringBuffer message = new StringBuffer();
1160        message.append("<html><body><h1> Bad Request (code "
1161                + HttpServletResponse.SC_BAD_REQUEST + ")</h1>\n");
1162        message.append("<p>Expected ");
1163        message.append(what);
1164        message.append(" of the form: ");
1165        if (expectedType instanceof RecordType) {
1166            _recordToURLString((RecordType) expectedType, message);
1167        } else {
1168            message.append(expectedType.toString());
1169        }
1170        message.append("</p><p>Got: ");
1171        _recordToURLString((RecordType) record.getType(), message);
1172        message.append("</p></body></html>");
1173
1174        _response.response = message.toString();
1175
1176        notifyAll();

1179    /** Issue a response indicating a server error and the intent to retry.
1180     *  This creates a response page that includes
1181     *  Javascript that will invoke the retry.
1182     *  FIXME: This only supports GET and POST requests currently.
1183     *  @param timeout The time to wait, in seconds
1184     */
1185    private void _respondWithRetryMessage(int timeout) {
1186        int timeoutValue = timeout;
1187
1188        if (timeoutValue < 1) {
1189            timeoutValue = 1;
1190        }
1191
1192        // Construct a response page.  The page will issue a second request to
1193        // the URL after a specified period of time.  The retry page content and
1194        // time are currently fixed.
1195        // TODO:  Allow dynamic content and timers for retry
1196
1197        _response = new HttpResponseItems();
1198        _response.statusCode = HttpServletResponse.SC_INTERNAL_SERVER_ERROR;
1199
1200        String ajax = "";
1201        if (_pendingRequest.method.equals("GET")) {
1202            ajax = "jQuery.get(\"" + _pendingRequest.requestURI + "\")\n"
1203                    + ".done(function(data) { \n " +
1204                    // Wrap result page with <div> </div> since an HTML page is not
1205                    // valid xml due to unclosed <!DOCTYPE HTML> tag
1206                    // jQuery has problems parsing otherwise
1207                    // http://www.sitepoint.com/secrets-selecting-elements-returned-jquery-ajax-response-strings/
1208                    "result = \"<div>\" + data + \"</div>\";"
1209                    + "jQuery(\"#contents\").html(jQuery(result).find(\"#contents\").html());"
1210                    + "\n });";
1211        } else if (_pendingRequest.method.equals("POST")) {
1212            StringBuffer parameters = new StringBuffer("{");
1213            if (_pendingRequest.parameters != null) {
1214                for (String label : _pendingRequest.parameters.labelSet()) {
1215                    // TODO:  Test if this works for strings
1216                    // I believe these require quotation marks around them
1217                    parameters.append(label + ": "
1218                            + _pendingRequest.parameters.get(label).toString()
1219                            + ",");
1220                }
1221
1222                // Erase the last , and add }
1223                if (parameters.length() > 0) {
1224                    parameters.deleteCharAt(parameters.length() - 1);
1225                    parameters.append('}');
1226                    ajax = "jQuery.post(\"" + _pendingRequest.requestURI
1227                            + "\", " + parameters.toString() + ")\n"
1228                            + ".done(function(data) { \n "
1229                            + "result = \"<div>\" + data + \"</div>\";"
1230                            + "jQuery(\"#contents\").html(jQuery(result).find(\"#contents\").html());"
1231                            + "\n });";
1232                } else {
1233                    ajax = "jQuery.post(\"" + _pendingRequest.requestURI
1234                            + "\")\n" + ".done(function(data) { \n "
1235                            + "result = \"<div>\" + data + \"</div>\";"
1236                            + "jQuery(\"#contents\").html(jQuery(result).find(\"#contents\").html());"
1237                            + "\n });";
1238                }
1239            }
1240        }
1241
1242        StringBuffer message = new StringBuffer();
1243
1244        message.append("<!DOCTYPE html>\n<html>\n<head> "
1245                + "<meta charset=\"UTF-8\">\n");
1246        message.append(
1247                "<script src=\"http://ajax.googleapis.com/ajax/libs/jquery/1.11.1/jquery.min.js\"></script>\n");
1248        message.append("<script>\n var count=" + timeoutValue + ";\n");
1249        message.append("var interval=setInterval(timer,1000);\nvar result;\n");
1250        message.append("function timer() {\n " + "count=count-1;\n"
1251                + "jQuery(\"#countdown\").html(count+1);\n"
1252                + "if (count <= 0) {\n" + "clearInterval(interval);\n"
1253                + ajax.toString() + "\n}\n}\n");
1254        message.append(
1255                "jQuery(document).ready(function() {\n" + "timer();\n});");
1256        message.append("</script></head>\n");
1257        message.append("<body><div id=\"contents\"> \n"
1258                + "<h1> Internal Server Error (code "
1259                + HttpServletResponse.SC_INTERNAL_SERVER_ERROR + ")</h1>\n");
1260        message.append("<div> Retrying in <div id=\"countdown\">" + timeoutValue
1261                + "</div></div></div>\n");
1262        message.append("</body></html>");
1263
1264        _response.response = message.toString();
1265
1266        notifyAll();
1267    }

1320    /** A servlet providing implementations of get and post.
1321     *  The way this servlet works is that when a get or post
1322     *  HTTP request is received, it records the properties of
1323     *  the request (the URI and parameters) and requests a
1324     *  firing of the enclosing actor at model time equal to
1325     *  the time elapsed since the start of execution of the model.
1326     *  The thread making the get or post request then suspends,
1327     *  giving the model a chance to execute. When the model has
1328     *  determined what the response to the request should be,
1329     *  it notifies this servlet thread, which then completes
1330     *  the handling of the request, sending back the response
1331     *  that has been provided by the enclosing actor.
1332     *
1333     *  <p>See <a href="http://wiki.eclipse.org/Jetty/Tutorial/Embedding_Jetty">http://wiki.eclipse.org/Jetty/Tutorial/Embedding_Jetty</a>.</p>
1334     */
1335    @SuppressWarnings("serial")
1336    protected class ActorServlet extends HttpServlet {
1337
1338        /** Handle an HTTP get request by creating a web page as the HTTP
1339         *  response.
1340         *  NOTE: This method is synchronized, and the lock is _not_ released
1341         *  while waiting for the response. This strategy helps prevent a pending
1342         *  get request from overlapping with a pending put request. The second
1343         *  request will be postponed until the first has been completely handled.
1344         *
1345         *  @param request The HTTP get request.
1346         *  @param response The HTTP response to write to.
1347         *  @exception ServletException  If there is a problem reading from the
1348         *  servlet request or other servlet problem
1349         *  @exception IOException  If there is a problem writing to the servlet
1350         *  response
1351         */
1352        @Override
1353        protected synchronized void doGet(HttpServletRequest request,
1354                HttpServletResponse response)
1355                throws ServletException, IOException {
1356            _handleRequest(request, response, "GET");
1357        }
1358
1359        /** Handle an HTTP post request by creating a web page as the HTTP
1360         *  response.
1361         *  NOTE: This method is synchronized, and the lock is _not_ released
1362         *  while waiting for the response. This strategy helps prevent a pending
1363         *  get request from overlapping with a pending put request. The second
1364         *  request will be postponed until the first has been completely handled.
1365         *  @param request The HTTP get request.
1366         *  @param response The HTTP response to write to.
1367         *  @exception ServletException  If there is a problem reading from the
1368         *  servlet request or other servlet problem
1369         *  @exception IOException  If there is a problem writing to the servlet
1370         *  response
1371         */
1372        @Override
1373        protected synchronized void doPost(HttpServletRequest request,
1374                HttpServletResponse response)
1375                throws ServletException, IOException {
1376            _handleRequest(request, response, "POST");
1377        }
1378    }
