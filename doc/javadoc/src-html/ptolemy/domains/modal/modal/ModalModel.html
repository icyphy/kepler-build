071/**
072 This is a typed composite actor designed to be a modal model.
073 Inside the modal model is a finite-state machine controller, and
074 inside each state in the FSM is a refinement model. To use this
075 actor, just drag it into a model, and look inside to start constructing
076 the controller.  You may add ports to get inputs and outputs, and
077 add states to the controller.  You may add one or more refinements
078 to a state (each of these refinements will be executed when this
079 actor is executed).  Each refinement is required to have its own
080 director, so you will need to choose a director.
081 <p>
082 The controller is a finite-state machine (FSM), which consists of
083 states and transitions.  One of the states is an initial state.
084 When this actor executes, if the current state has a refinement,
085 then that refinement is executed.  Then the guards on all the outgoing
086 transitions of the current state are evaluated, and if one of those
087 guards is true, then the transition is taken.  Taking the transition
088 means that the actions associated with the transition are executed
089 (which can result in producing outputs), and the new current state is
090 the state at the destination of the transition.  It is an error if
091 more than one of the guards evaluates to true.
092 <p>
093 To add a state, click on a state button in the toolbar, or drag
094 in a state from the library at the left.  To add a transition,
095 position the mouse over the source state, hold the control button,
096 and drag to the destination state.  The destination state may be
097 the same state, in which case the transition is used simply to
098 execute its actions.
099 <p>
100 Adding or removing ports in this actor results in the same ports appearing
101 or disappearing in the FSM controller and in each of the refinements.
102 Similarly, adding or removing ports in the controller or in the
103 refinements results in this actor and the other refinements
104 reflecting the same change to the ports.  That is, this actor,
105 the controller, and the refinments all contain the same ports.
106 <p>
107 There is one subtlety regarding ports however.  If you add an
108 output port to a refinement, then the corresponding port in the
109 controller will be both an input and an output.  The reason for
110 this is that the controller can access the results of executing
111 a refinement in order to choose a transition.
112 <p>
113 This class is designed to work closely with ModalController and
114 Refinement, since changes to ports can be initiated in this class
115 or in those. It works with continuous-time as well as discrete-time
116 models.
117 <p>
118 By default, this actor has a conservative causality interface,
119 which examines the FSMActor controller and all the refinements
120 and defines input/output dependencies that are the oPlus combination
121 of all their dependencies. If
122 the <i>stateDependentCausality</i> is false (the default),
123 then this causality interface in conservative and valid in all
124 states. If it is true, then the causality interface will show
125 different input/output dependencies depending on the state.
126 In each state, only the controller and the current refinement
127 will be considered, and in the controller, only the outgoing
128 transitions from the current state will be considered.
129 <p>
130 This class also fulfills the CTEventGenerator interface so that
131 it can report events generated inside.
132
133 @see ModalController
134 @see Refinement
135 @author Edward A. Lee and Shanna-Shaye Forbes
136 @version $Id$
137 @since Ptolemy II 8.0
138 @Pt.ProposedRating Red (eal)
139 @Pt.AcceptedRating Red (reviewmoderator)
140 */

199    /** React to a change of the director or other property. */
200    @Override
201    public void attributeChanged(Attribute attribute)
202            throws IllegalActionException {
203        if (attribute == directorClass) {
204            // We should change the director only if the current
205            // director is not of the right class.
206            Director director = getDirector();
207            String className = directorClass.stringValue();
208
209            if (director == null
210                    || !director.getClass().getName().equals(className)) {
211                // Check the class name to get immediate feedback
212                // to the user.
213                try {
214                    Class.forName(className);
215                } catch (ClassNotFoundException e) {
216                    if (className.equals(
217                            "ptolemy.domains.modal.kernel.FSMDirector")) {
218                        className = "ptolemy.domains.modal.kernel.FSMDirector";
219                        try {
220                            Class.forName(className);
221                        } catch (ClassNotFoundException e2) {
222                            throw new IllegalActionException(this, null, e2,
223                                    "Invalid directorClass. \"" + className
224                                            + "\".");
225                        }
226                    } else {
227                        throw new IllegalActionException(this, null, e,
228                                "Invalid directorClass. \"" + className
229                                        + "\".");
230                    }
231                }
232
233                final String finalClassName = className;
234                // NOTE: Creating a new director has to be done in a
235                // change request.
236                ChangeRequest request = new ChangeRequest(this,
237                        "Create a new director") {
238                    @Override
239                    protected void _execute() throws Exception {
240                        Director director = getDirector();
241
242                        // Contruct a new director
243                        //Class newDirectorClass = Class.forName(directorClass
244                        //        .stringValue());
245                        Class newDirectorClass = Class.forName(finalClassName);
246                        Constructor newDirectorConstructor = newDirectorClass
247                                .getConstructor(new Class[] {
248                                        CompositeEntity.class, String.class });
249                        Director newDirector = (Director) newDirectorConstructor
250                                .newInstance(new Object[] { ModalModel.this,
251                                        uniqueName("_Director") });
252
253                        // The director should not be persistent.
254                        newDirector.setPersistent(false);
255                        try {
256                            StringAttribute newControllerName = (StringAttribute) newDirector
257                                    .getAttribute("controllerName");
258                            newControllerName.setExpression("_Controller");
259                        } catch (Exception e) {
260                            throw new IllegalActionException("Director class \""
261                                    + newDirectorClass + "\" cannot be used "
262                                    + "because it does not have a "
263                                    + "\"controllerName\" attribute.");
264                        }
265
266                        if (director != null
267                                && director.getContainer() == ModalModel.this) {
268                            // Delete the old director.
269                            director.setContainer(null);
270                        }
271
272                        // Check whether the modal controller needs to
273                        // support multirate firing.
274                        Director executiveDirector = getExecutiveDirector();
275
276                        if (executiveDirector != null) {
277                            // Need both the executive director and the local director to
278                            // support multirate firing in order for this to work.
279                            if (newDirector.supportMultirateFiring()
280                                    && executiveDirector
281                                            .supportMultirateFiring()) {
282                                getController().setSupportMultirate(true);
283                            }
284                        }
285                    }
286                };
287                requestChange(request);
288            }
289        } else {
290            super.attributeChanged(attribute);
291        }
292    }

345    /** Override the base class to remove any unused refinements
346     *  before exporting.
347     *  @param output The output stream to write to.
348     *  @param depth The depth in the hierarchy, to determine indenting.
349     *  @param name The name to use in the exported MoML.
350     *  @exception IOException If an I/O error occurs.
351     *  @see ptolemy.kernel.util.MoMLExportable
352     */
353    @Override
354    public void exportMoML(Writer output, int depth, String name)
355            throws IOException {
356        try {
357            HashSet<TypedActor> activeRefinements = new HashSet<TypedActor>();
358            for (State state : _controller.entityList(State.class)) {
359                TypedActor refinements[] = state.getRefinement();
360                if (refinements != null) {
361                    for (TypedActor refinement : refinements) {
362                        activeRefinements.add(refinement);
363                    }
364                }
365            }
366            List<Transition> transitions = _controller.relationList();
367            for (Transition transition : transitions) {
368                TypedActor refinements[] = transition.getRefinement();
369                if (refinements != null) {
370                    for (TypedActor refinement : refinements) {
371                        activeRefinements.add(refinement);
372                    }
373                }
374            }
375            for (TypedActor actor : entityList(Refinement.class)) {
376                // FindBugs: Using pointer equality to compare _controller (a FSMActor)
377                // with actor, which is a Refinement)
378                if (!activeRefinements
379                        .contains(actor) /*&& actor != _controller*/) {
380                    if (MessageHandler.yesNoQuestion(
381                            "Unused state refinement in modal model: "
382                                    + actor.getFullName() + ". Remove it?")) {
383                        ((ComponentEntity) actor).setContainer(null);
384                    }
385                }
386            }
387        } catch (KernelException ex) {
388            // This should not happen. Ignore and continue the export.
389            System.out.println(ex);
390        }
391        super.exportMoML(output, depth, name);
392    }

424    /** Get representation of dependencies between input ports and
425     *  output ports.
426     *  @return A representation of the dependencies between input ports
427     *   and output ports.
428     */
429    @Override
430    public CausalityInterface getCausalityInterface() {
431        try {
432            boolean stateDependentCausality = ((BooleanToken) _controller.stateDependentCausality
433                    .getToken()).booleanValue();
434            if (!stateDependentCausality) {
435                return super.getCausalityInterface();
436            }
437            // Causality is state dependent if we get here.
438            // Return the causality interface of the
439            // controller FSMActor composed with
440            // the current refinement causality interfaces.
441            if (_causalityInterfacesVersions == null) {
442                _causalityInterfacesVersions = new HashMap<State, Long>();
443                _causalityInterfaces = new HashMap<State, MirrorCausalityInterface>();
444            }
445            State currentState = _controller.currentState();
446            Long version = _causalityInterfacesVersions.get(currentState);
447            MirrorCausalityInterface causality = _causalityInterfaces
448                    .get(currentState);
449            if (version == null || causality == null
450                    || version.longValue() != workspace().getVersion()) {
451                // Need to create or update a causality interface for the current state.
452                CausalityInterface controllerCausality = _controller
453                        .getCausalityInterface();
454                causality = new MirrorCausalityInterface(this,
455                        controllerCausality);
456                Actor[] refinements = currentState.getRefinement();
457                if (refinements != null) {
458                    for (Actor refinement : refinements) {
459                        CausalityInterface refinementCausality = refinement
460                                .getCausalityInterface();
461                        causality.composeWith(refinementCausality);
462                    }
463                }
464                _causalityInterfaces.put(currentState, causality);
465                _causalityInterfacesVersions.put(currentState,
466                        Long.valueOf(workspace().getVersion()));
467            }
468            return causality;
469        } catch (IllegalActionException e) {
470            throw new InternalErrorException(e);
471        }
472    }

504    /** Initialize the mode controller and all the refinements by
505     * calling the initialize() method in the super class. Build the
506     * local maps for receivers. Suspend all the refinements of states
507     * that are not the current state.
508     *
509     * @exception IllegalActionException If thrown by the initialize()
510     *                method of the super class, or can not find mode
511     *                controller, or can not find refinement of the
512     *                current state.

514    @Override
515    public void initialize() throws IllegalActionException {
516        // Reset local receivers here before the initialize method
517        // of the director so that any initial outputs that
518        // the FSM controller or refinements might produce in their initialize methods
519        // are the only outputs produced by initialize.
520        Director director = getDirector();
521        if (director instanceof FSMDirector) {
522            ((FSMDirector) director).resetOutputReceivers();
523        }
524        super.initialize();
525    }

527    /** Create a new port with the specified name in this entity, the
528     *  controller, and all the refinements.  Link these ports so that
529     *  if the new port is set to be an input, output, or multiport, then
530     *  the change is mirrored in the other ports.  The new port will be
531     *  an instance of ModalPort, which extends TypedIOPort.
532     *  This method is write-synchronized on the workspace, and increments
533     *  its version number.
534     *  @param name The name to assign to the newly created port.
535     *  @return The new port.
536     *  @exception NameDuplicationException If the entity already has a port
537     *   with the specified name.
538     */
539    @Override
540    public Port newPort(String name) throws NameDuplicationException {
541        try {
542            _workspace.getWriteAccess();
543
544            ModalPort port = new ModalPort(this, name);
545
546            // Create mirror ports.
547            Iterator entities = entityList().iterator();
548
549            while (entities.hasNext()) {
550                Entity entity = (Entity) entities.next();
551
552                if (entity instanceof RefinementActor) {
553                    if (entity.getPort(name) == null) {
554                        try {
555                            ((RefinementActor) entity).setMirrorDisable(1);
556                            entity.newPort(name);
557                        } finally {
558                            ((RefinementActor) entity).setMirrorDisable(0);
559                        }
560                    }
561                }
562            }
563
564            return port;
565        } catch (IllegalActionException ex) {
566            // This exception should not occur, so we throw a runtime
567            // exception.
568            throw new InternalErrorException(
569                    "ModalModel.newPort: Internal error: " + ex.getMessage());
570        } finally {
571            _workspace.doneWriting();
572        }
573    }

583    // Initialize the model.
584    private void _init()
585            throws IllegalActionException, NameDuplicationException {
586        // The base class identifies the class name as TypedCompositeActor
587        // irrespective of the actual class name.  We override that here.
588        setClassName("ptolemy.domains.modal.modal.ModalModel");
589
590        stateDependentCausality = new Parameter(this,
591                "stateDependentCausality");
592        stateDependentCausality.setTypeEquals(BaseType.BOOLEAN);
593        stateDependentCausality.setExpression("false");
594
595        // Create a default modal controller.
596        // NOTE: It would be much nicer if the director created the
597        // controller it likes (or has it configured) and returned it
598        // (zk 2002/09/11)
599        _controller = new ModalController(this, "_Controller");
600        // Whether the controller does conservative analysis or not should
601        // depend on the parameter of this actor.
602        _controller.stateDependentCausality
603                .setExpression("stateDependentCausality");
604
605        // configure the directorClass parameter
606        directorClass = new StringParameter(this, "directorClass");
607
608        // Set the director to the default. Note that doing
609        // this manually rather than in attributeChanged() prevents
610        // attributeChanged() from issuing a change request
611        // (because the director class matches the default).
612        // Issuing a change request during construction is
613        // problematic because it causes the Vergil library
614        // to close when you first open a sublibrary containing
615        // an instance of ModalModel.
616        FSMDirector defaultFSMDirector = new FSMDirector(this, "_Director");
617        defaultFSMDirector.controllerName.setExpression("_Controller");
618
619        // NOTE: If there is a container for this ModalModel, and it
620        // has a director, then we get the default value from that
621        // director, and also get a list of suggested values.
622        Director executiveDirector = getExecutiveDirector();
623
624        if (executiveDirector != null) {
625            // FIXME: Better solution is to override the returned
626            // list of choices in the parameter class.
627            String[] suggestions = executiveDirector
628                    .suggestedModalModelDirectors();
629
630            for (int i = 0; i < suggestions.length; i++) {
631                suggestions[i] = suggestions[i].replace("domains.fsm.",
632                        "domains.modal.");
633                directorClass.addChoice(suggestions[i]);
634                if (i == 0) {
635                    directorClass.setExpression(suggestions[i]);
636                }
637            }
638        } else {
639            // If there is no executive director. Use the default director.
640            // This happens when vergil starts, and when a modal model is
641            // dropped into a blank editor. Model designers need to configure
642            // it if FSMDirector is not the desired director.
643            directorClass
644                    .setExpression("ptolemy.domains.modal.kernel.FSMDirector");
645        }
646
647        // Create a more reasonable default icon.
648        _attachText("_iconDescription",
649                "<svg>\n" + "<rect x=\"-30\" y=\"-20\" width=\"60\" "
650                        + "height=\"40\" style=\"fill:red\"/>\n"
651                        + "<rect x=\"-28\" y=\"-18\" width=\"56\" "
652                        + "height=\"36\" style=\"fill:lightgrey\"/>\n"
653                        + "<ellipse cx=\"0\" cy=\"0\""
654                        + " rx=\"15\" ry=\"10\"/>\n"
655                        + "<circle cx=\"-15\" cy=\"0\""
656                        + " r=\"5\" style=\"fill:white\"/>\n"
657                        + "<circle cx=\"15\" cy=\"0\""
658                        + " r=\"5\" style=\"fill:white\"/>\n" + "</svg>\n");
659
660    }
