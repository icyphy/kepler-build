028package ptolemy.domains.pn.kernel.event;
029
030//////////////////////////////////////////////////////////////////////////
031//// PNProcessListener
032
033/**
034 An PNProcessListener is able to receive PNProcessEvents that are issued
035 during the execution of a process by a ProcessThread or director in PN.
036 In general, an object that implements this interface will probably be a front
037 end such as a execution visualization tool for
038 the Ptolemy II system, or an object that is communicating with a front end.
039 The events are issued only when the event actually occurs, not when it is
040 requested.   For example: A process receives a call to pause(), but the
041 process may pause some time after the pause was requested. The processPaused()
042 method will not be called until the process actually pauses and corresponding
043 thread is suspended.
044
045 @author Mudit Goel
046 @version $Id$
047 @since Ptolemy II 0.3
048 @Pt.ProposedRating Yellow (mudit)
049 @Pt.AcceptedRating Red
050 */
051public interface PNProcessListener {
052    /** Called to report that the execution of a process finished. The
053     *  wrapup sequence may or may not have completed normally.   The
054     *  execution event will contain a reference to the actor corresponding
055     *  to the process that finished and the reason for finishing.
056     *
057     *  @param event A PNProcessEvent that contains a reference to an
058     *  actor.
059     */
060    public void processFinished(PNProcessEvent event);
061
062    /** Called to report that a process has changed its state (i.e. started,
063     *  or blocked or unblocked, etc.). The PNProcessEvent
064     *  will contain a reference to the actor corresponding to the process.
065     *  The event will also indicate the new state and blocking cause, etc.
066     *
067     *  @param event A PNProcessEvent that contains a reference to an actor.
068     */
069    public void processStateChanged(PNProcessEvent event);
070}
