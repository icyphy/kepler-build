077/**
078 An application that reads one or more
079 files specified on the command line, or instantiates one or
080 more Java classes specified by the -class option.
081 If one of these files is an XML file that defines a Configuration, or one
082 of the classes is an instance of Configuration, then
083 all subsequent files will be read by delegating to the Configuration,
084 invoking its openModel() method.  A command-line file is assumed to be
085 a MoML file or a file that can be opened by the specified configuration.
086 <p>For example, this command uses the HyVisual configuration to
087 open a model:
088 <pre>
089 $PTII/bin/moml $PTII/ptolemy/configs/hyvisual/configuration.xml $PTII/ptolemy/domains/ct/demo/StickyMasses/StickyMasses.xml
090 </pre>
091 <p>
092 If a Ptolemy model is instantiated on the command line, either
093 by giving a MoML file or a -class argument, then parameters of that
094 model can be set on the command line.  The syntax is:
095 <pre>
096 $PTII/bin/ptolemy <i>modelFile.xml</i> -<i>parameterName</i> <i>value</i>
097 </pre>
098 where <i>parameterName</i> is the name of a parameter relative to
099 the top level of a model or the director of a model.  For instance,
100 if foo.xml defines a toplevel entity named <code>x</code> and
101 <code>x</code> contains an entity named <code>y</code> and a
102 parameter named <code>a</code>, and <code>y</code> contains a
103 parameter named <code>b</code>, then:
104 <pre>
105 $PTII/bin/ptolemy foo.xml -a 5 -y.b 10
106 </pre>
107 would set the values of the two parameters.
108
109 <p>Note that strings need to have double quotes converted to
110 <code>&amp;quot;</code> so to set a parameter named <code>c</code>
111 to the string <code>"bar"</code> it might be necessary to do
112 something like:</p>
113 <pre>
114 $PTII/bin/ptolemy foo.xml -a 5 -y.b 10 -c "&amp;quot;bar&amp;quot;"
115 </pre>
116 <p>The <code>&amp;quot;</code> is necessary to convert the double quote
117 to something safe to in an XML file.  The backslashes are necessary
118 to protect the <code>&amp;</code> and <code>;</code> from the shell
119 in the shell script.</p>
120
121 <p>Note that the ptolemy.actor.parameters.ParameterSet attribute is
122 a better way to set parameters at run time.  ParameterSet is
123 an attribute that reads multiple values from a file and sets
124 corresponding parameters in the container.</p>
125
126 <p>The -class option can be used to specify a Java class to be loaded.
127 The named class must have a constructor that takes a Workspace
128 as an argument.
129 In the example below, $PTII/ptolemy/domains/sdf/demo/Butterfly/Butterfly.java
130 is a class that has a constructor Butterfly(Workspace).
131 <pre>
132 $PTII/bin/ptolemy -class ptolemy.domains.sdf.demo.Butterfly.Butterfly
133 </pre>
134 Note that -class is very well tested now that we have use MoML
135 for almost all models.
136
137 <p>
138 Derived classes may provide default configurations. In particular, the
139 protected method _createDefaultConfiguration() is called before any
140 arguments are processed to provide a default configuration for those
141 command-line command-line arguments.  In this base class,
142 that method returns null, so no default configuration is provided.
143 <p>
144 If no arguments are given at all, then a default configuration is instead
145 obtained by calling the protected method _createEmptyConfiguration().
146 In this base class, that method also returns null,
147 so calling this with no arguments will not be very useful.
148 No configuration will be created and no models will be opened.
149 Derived classes can specify a configuration that opens some
150 welcome window, or a blank editor.
151
152 <p> This class read the following parameters from the configuration:
153 <dl>
154 <dt> <code>_applicationInitializer</code>
155 <dd> A StringParameter that names a class to be instantiated.
156 Kepler uses this parameter to instantiate KeplerInitializer:
157 <pre>
158 &gt;property name="_applicationInitializer"
159         class="ptolemy.data.expr.StringParameter"
160         value="org.kepler.gui.KeplerInitializer"/&lt;
161</pre>
162
163 </dl>
164
165 <p>To run this class from the command line without any of the
166 Ptolemy-specific scripts, try:</p>
167<pre>
168java -classpath $PTII ptolemy.actor.gui.ConfigurationApplication \
169   -run20x $PTII/ptolemy/configs/full/configuration.xml \
170   $PTII/ptolemy/actor/lib/test/auto/Ramp1.xml
171</pre>

219     *  and reading files that are specified.
220     *  @param basePath The basePath to look for configurations
221     *  in, usually "ptolemy/configs", but other tools might
222     *  have other configurations in other directories
223     *  @param args The command-line arguments.
224     *  @param messageHandler The message handler.
225     *  @param errorHandler the MoML error handler.
226     *  @exception Exception If command line arguments have problems.
227     */
228    public ConfigurationApplication(String basePath, String[] args,
229            MessageHandler messageHandler, ErrorHandler errorHandler)
230            throws Exception {
231        this();
232
233        _initializeApplication();
234
235        try {
236            StringUtilities.addPtolemyLibraryDirectoryToJavaLibraryPath();
237        } catch (Throwable throwable) {
238            // Java 12 no longer has ClassLoader.usr_paths.
239            System.err.println("ConfigurationApplication(): Could not add the ptolemy "
240                               + "library directory to the Java library path: "
241                               + throwable);
242            throwable.printStackTrace();
243        }
244        _basePath = basePath;
245
246        // Create a parser to use.
247        _parser = new MoMLParser();
248
249        MoMLParser.setErrorHandler(errorHandler);
250
251        // We set the list of MoMLFilters to handle Backward Compatibility.
252        MoMLParser.setMoMLFilters(BackwardCompatibility.allFilters());
253
254        // 2/03: Moved the setMessageHandler() to before parseArgs() so
255        // that if we get an error in parseArgs() we will get a graphical
256        // stack trace.   Such an error could be caused by specifying a model
257        // as a command line argument and the model has an invalid parameter.
258        MessageHandler.setMessageHandler(messageHandler);
259
260        // Even if the user is set up for foreign locale, use the US locale.
261        // This is because certain parts of Ptolemy (like the expression
262        // language) are not localized.
263        // FIXME: This is a workaround for the locale problem, not a fix.
264        // FIXME: In March, 2001, Johan Ecker writes
265        // Ptolemy gave tons of exception when started on my laptop
266        // which has Swedish settings as default. The Swedish standard
267        // for floating points are "2,3", i.e. using a comma as
268        // delimiter. However, I think most Swedes are adaptable and
269        // do not mind using a dot instead since this more or less has
270        // become the world standard, at least in engineering. The
271        // problem is that I needed to change my global settings to
272        // start Ptolemy and this is quite annoying. I guess that the
273        // expression parser should just ignore the delimiter settings
274        // on the local computer and always use dot, otherwise Ptolemy
275        // will crash using its own init files.
276        try {
277            java.util.Locale.setDefault(java.util.Locale.US);
278        } catch (java.security.AccessControlException accessControl) {
279            // FIXME: If the application is run under Web Start, then this
280            // exception will be thrown.
281        }
282
283        try {
284            _parseArgs(args);
285
286            if (_statistics && !_run) {
287                Iterator models = models().iterator();
288
289                while (models.hasNext()) {
290                    NamedObj model = (NamedObj) models.next();
291                    if (model instanceof CompositeEntity) {
292                        System.out.println(
293                                "Statistics for " + model.getFullName());
294                        System.out.println(
295                                ((CompositeEntity) model).statistics(null));
296                    }
297                }
298                if (_exit) {
299                    StringUtilities.exit(0);
300                }
301            }
302
303            // Run if -run argument was specified.
304            if (_run) {
305                if (_printPDF) {
306                    // Need to set background
307                    PtolemyPreferences preferences = PtolemyPreferences
308                            .getPtolemyPreferencesWithinConfiguration(
309                                    _configuration);
310                    preferences.backgroundColor
311                            .setExpression("{1.0, 1.0, 1.0, 1.0}");
312
313                }
314                if (_run20x) {
315                    for (int i = 1; i <= 20; i++) {
316                        // Use Manager.execute()
317                        _runModels(false);
318                    }
319                } else {
320                    // Use Manager.startRun() and run the model in
321                    // a new thread.
322                    _runModels(true);
323                }
324
325                if (_exit) {
326                    // In vergil, this gets called in the
327                    // swing thread, which hangs everything
328                    // if we call waitForFinish() directly.
329                    // So instead, we create a new thread to
330                    // do it.
331                    Thread waitThread = new Thread() {
332                        @Override
333                        public void run() {
334                            waitForFinish();
335                            if (_printPDF) {
336                                try {
337                                    _printPDF();
338                                } catch (Exception ex) {
339                                    ex.printStackTrace();
340                                }
341                            }
342                            StringUtilities.exit(0);
343                        }
344                    };
345
346                    // Note that we start the thread here, which could
347                    // be risky when we subclass, since the thread will be
348                    // started before the subclass constructor finishes (FindBugs)
349                    waitThread.start();
350                }
351            } else {
352                if (_printPDF) {
353                    _printPDF();
354                }
355            }
356
357        } catch (Throwable ex) {
358            // Make sure that we do not eat the exception if there are
359            // problems parsing.  For example, "ptolemy -FOO bar bif.xml"
360            // will crash if bar is not a variable.  Instead, use
361            // "ptolemy -FOO \"bar\" bif.xml"
362            throwArgsException(ex, args);
363        }
364    }

369    /** Close the model without saving or exiting.
370     *
371     *  <p>The caller of this method should be in the Swing Event Thread.
372     *  Typically, this is done with code like:</p>
373     *  <pre>
374     *   Runnable openModelAction = new Runnable() {
375     *       public void run() {
376     *           try {
377     *               ConfigurationApplication.closeModel(model[0])
378     *           } catch (Exception ex) {
379     *               throw new RuntimeException(ex);
380     *           }
381     *       }
382     *   };
383     *   SwingUtilities.invokeAndWait(openModelAction);
384     *  </pre>
385     *  <p>This method is primarily used for testing.</p>
386     *  @param model The TypedCompositeActor to be closed.
387     *  Typically, this comes from {@link #openModel(String)}.
388     *  @exception IllegalActionException If the model cannot be closed.
389     *  @see #openModel(String)
390     *  @exception NameDuplicationException If the model cannot be closed.
391     *  @see #openModel(String)
392     */
393    public static void closeModelWithoutSavingOrExiting(CompositeEntity model)
394            throws IllegalActionException, NameDuplicationException {
395        Effigy effigy = Configuration.findEffigy(model.toplevel());
396        // Avoid being prompted for save.
397        effigy.setModified(false);
398
399        // Avoid calling System.exit().
400        String previousPropertyValue = StringUtilities
401                .getProperty("ptolemy.ptII.doNotExit");
402        System.setProperty("ptolemy.ptII.doNotExit", "true");
403        try {
404            // FIXME: are all these necessary?
405            effigy.closeTableaux();
406
407            // Avoid a leaking Configuration.
408            // This might not be safe, if we had a WeakLinkedList, we
409            // could use make Configuration._configuration a WeakLinkedList.
410            // Or, perhaps effigy.closeTableaux() should do this.
411            Configuration configuration = (Configuration) Configuration
412                    .findEffigy(model.toplevel()).toplevel();
413            configuration.removeConfiguration(configuration);
414
415            model.setContainer(null);
416            MoMLParser.purgeAllModelRecords();
417        } finally {
418            System.setProperty("ptolemy.ptII.doNotExit", previousPropertyValue);
419        }
420    }

429    public static File[] configurationDirectories(String basePath)
430            throws IOException, URISyntaxException {
431        URI configurationURI = new URI(
432                specToURL(basePath).toExternalForm().replaceAll(" ", "%20"));
433        File configurationDirectory = new File(configurationURI);
434        ConfigurationFilenameFilter filter = new ConfigurationFilenameFilter();
435        File[] configurationDirectories = configurationDirectory
436                .listFiles(filter);
437        return configurationDirectories;
438    }

440    /** Return the full configuration directory or, if the full configuration
441     *  directory is not found, then the first configuration directory.
442     *  @return the possible configuration directories.
443     *  @exception IOException If the basePath cannot be found.
444     *  @exception URISyntaxException If the URI of the basePath is incorrect.
445     */
446    public static File configurationDirectoryFullOrFirst()
447            throws IOException, URISyntaxException {
448        File[] configurationDirectories = configurationDirectories(
449                "ptolemy/configs");
450        if (configurationDirectories.length < 1) {
451            throw new IOException(
452                    "Could not find any configurations in ptolemy/configs");
453        }
454        File configurationDirectory = configurationDirectories[0];
455        int i;
456        for (i = 0; i < configurationDirectories.length; i++) {
457            if (configurationDirectories[i].toString()
458                    .endsWith("configs/full")) {
459                configurationDirectory = configurationDirectories[i];
460            }
461        }
462        return configurationDirectory;
463    }

541    /** Return a list of the Ptolemy II models that were created by processing
542     *  the command-line arguments.
543     *  @return A list of instances of NamedObj.
544     */
545    public List<NamedObj> models() {
546        LinkedList<NamedObj> result = new LinkedList<NamedObj>();
547
548        if (_configuration == null) {
549            return result;
550        }
551
552        ModelDirectory directory = (ModelDirectory) _configuration
553                .getEntity(Configuration._DIRECTORY_NAME);
554        if (directory == null) {
555            throw new InternalErrorException("Failed to get the "
556                    + "model directory? This can happen "
557                    + "in a headless environment when the model attempts to "
558                    + "interact with the graphical display.  "
559                    + "It can also happen when the model fails to parse "
560                    + "because of a missing class.");
561        }
562        Iterator effigies = directory.entityList().iterator();
563
564        while (effigies.hasNext()) {
565            Effigy effigy = (Effigy) effigies.next();
566
567            if (effigy instanceof PtolemyEffigy) {
568                NamedObj model = ((PtolemyEffigy) effigy).getModel();
569                result.add(model);
570            }
571        }
572
573        return result;
574    }

723    /** Read a Configuration from the URL given by the specified string.
724     *  The URL may absolute, or relative to the Ptolemy II tree root,
725     *  or in the classpath.  To convert a String to a URL suitable for
726     *  use by this method, call specToURL(String).
727     *  <p>If there is an _applicationInitializer parameter, then
728     *  instantiate the class named by that parameter.  The
729     *  _applicationInitializer parameter contains a string that names
730     *  a class to be initialized.
731     *  <p>If the configuration has already been read in, then the old
732     *  configuration will be deleted.  Note that this may exit the application.
733     *  @param specificationURL A string describing a URL.
734     *  @return A configuration.
735     *  @exception Exception If the configuration cannot be opened, or
736     *   if the contents of the URL is not a configuration.
737     */
738    public static Configuration readConfiguration(URL specificationURL)
739            throws Exception {
740        if (_initialSpecificationURI == null) {
741            _initialSpecificationURI = specificationURL.toURI();
742        }
743        MoMLParser parser = new MoMLParser();
744        parser.reset();
745
746        Configuration configuration = null;
747        Exception cause = null;
748        try {
749            configuration = (Configuration) parser.parse(specificationURL,
750                    specificationURL);
751        } catch (Exception ex) {
752            cause = ex;
753            ex.printStackTrace();
754        }
755
756        if (configuration == null) {
757            NullPointerException exception = new NullPointerException(
758                    "Failed to find configuration in " + specificationURL);
759            if (cause != null) {
760                exception.initCause(cause);
761            }
762            throw exception;
763        }
764        // If the toplevel model is a configuration containing a directory,
765        // then create an effigy for the configuration itself, and put it
766        // in the directory.
767        ComponentEntity directory = configuration.getDirectory();
768
769        if (directory != null) {
770            PtolemyEffigy effigy = null;
771            try {
772                effigy = new PtolemyEffigy((ModelDirectory) directory,
773                        configuration.getName());
774                // Mark this to be a system effigy so it is not deleted
775                // when an effigy it contains is deleted.
776                effigy.setSystemEffigy(true);
777            } catch (NameDuplicationException ex) {
778                // Try deleting the old configuration
779                PtolemyEffigy oldEffigy = (PtolemyEffigy) ((ModelDirectory) directory)
780                        .getEntity(configuration.getName());
781                oldEffigy.setContainer(null);
782                effigy = new PtolemyEffigy((ModelDirectory) directory,
783                        configuration.getName());
784            }
785
786            effigy.setModel(configuration);
787            effigy.identifier.setExpression(specificationURL.toExternalForm());
788        }
789
790        // If there is an _applicationInitializer parameter, then
791        // construct it.  The _applicationInitializer parameter contains
792        // a string that names a class to be initialized.
793        StringParameter applicationInitializerParameter = (StringParameter) configuration
794                .getAttribute("_applicationInitializer", Parameter.class);
795
796        if (applicationInitializerParameter != null) {
797            String applicationInitializerClassName = applicationInitializerParameter
798                    .stringValue();
799            try {
800                Class applicationInitializer = Class
801                        .forName(applicationInitializerClassName);
802                applicationInitializer.newInstance();
803            } catch (Throwable throwable) {
804                throw new Exception(
805                        "Failed to call application initializer " + "class \""
806                                + applicationInitializerClassName
807                                + "\".  Perhaps the configuration file \""
808                                + specificationURL + "\" has a problem?",
809                        throwable);
810            }
811        }
812
813        return configuration;
814    }

823    /** Given the name of a file or a URL, convert it to a URL.
824     *  This first attempts to do that directly by invoking a URL constructor.
825     *  If that fails, then it tries to interpret the spec as a file name
826     *  on the local file system.  If that fails, then it tries to interpret
827     *  the spec as a resource accessible to the class loader, which uses
828     *  the classpath to find the resource.  If that fails, then it throws
829     *  an exception.  The specification can give a file name relative to
830     *  current working directory, or the directory in which this application
831     *  is started up.
832     *  @param spec The specification.
833     *  @return the URL.
834     *  @exception IOException If it cannot convert the specification to
835     *   a URL.
836     */
837    public static URL specToURL(String spec) throws IOException {
838        // FIXME: There is a bit of a design flaw here because
839        // we open a stream to the url (which is probably expensive)
840        // and then close it.  The reason for opening the stream
841        // is that we want to be sure that the URL is valid,
842        // and if it is not, we check the local file system
843        // and the classpath.
844        // One solution would be to have a method that returned a
845        // URLConnection because we can open a stream with a
846        // URLConnection and still get the original URL if necessary
847        URL specURL = null;
848
849        try {
850            // First argument is null because we are only
851            // processing absolute URLs this way.  Relative
852            // URLs are opened as ordinary files.
853            specURL = new URL(null, spec);
854            // Make sure that the specURL actually exists
855            InputStream urlStream = specURL.openStream();
856            urlStream.close();
857            return specURL;
858        } catch (Exception ex) {
859            try {
860                // Try as a regular file
861                File file = new File(spec);
862
863                // Oddly, under Windows file.exists() might return even
864                // though the file does not exist if we changed user.dir.
865                // See
866                // http://forum.java.sun.com/thread.jsp?forum=31&thread=328939
867                // One hack is to convert to an absolute path first
868                File absoluteFile = file.getAbsoluteFile();
869
870                try {
871                    if (!absoluteFile.exists()) {
872                        throw new IOException(
873                                "File '" + absoluteFile + "' does not exist.");
874                    }
875                } catch (java.security.AccessControlException accessControl) {
876                    IOException exception = new IOException(
877                            "AccessControlException while " + "trying to read '"
878                                    + absoluteFile + "'");
879
880                    // IOException does not have a cause argument constructor.
881                    exception.initCause(accessControl);
882                    throw exception;
883                }
884
885                specURL = absoluteFile.getCanonicalFile().toURI().toURL();
886                //InputStream urlStream = specURL.openStream();
887                //urlStream.close();
888                return specURL;
889            } catch (Throwable throwable) {
890                try {
891                    // Try one last thing, using the classpath.
892                    // Need a class context, and this is a static method, so...
893                    // we can't use this.getClass().getClassLoader()
894                    // NOTE: There doesn't seem to be any way to convert
895                    // this a canonical name, so if a model is opened this
896                    // way, and then later opened as a file, the model
897                    // directory will think it has two different files.
898                    //Class refClass = Class.forName(
899                    //        "ptolemy.kernel.util.NamedObj");
900                    //specURL = refClass.getClassLoader().getResource(spec);
901                    // This works in Web Start, see
902                    // http://download.oracle.com/javase/1.5.0/docs/guide/javaws/developersguide/faq.html#211
903
904                    // Unfortunately, Thread.currentThread().getContextClassLoader() can
905                    // return null.  This happened when
906                    // $CLASSPATH/ptolemy/actor/lib/vertx/demo/TokenTransmissionTime/Sender.xml
907                    // failed and subsequent calls to openModelOrEntity() would
908                    // fail because the configuration could not be found.
909                    // So, we have our own getResource() that handles this.
910                    specURL = ClassUtilities.getResource(spec);
911
912                    if (specURL == null) {
913                        try {
914                            specURL = ClassUtilities.jarURLEntryResource(spec);
915                        } catch (IOException ex4) {
916                            // Ignore
917                        }
918                        if (specURL == null) {
919                            // If we have a jar URL, convert spaces to %20
920                            // so as to avoid multiple windows with the
921                            // same file.  Web Start needs this if the Web
922                            // Start cache is in a directory that has
923                            // spaces in the path, which is the default
924                            // under Windows.
925                            specURL = JNLPUtilities
926                                    .canonicalizeJarURL(new URL(spec));
927                            if (specURL == null) {
928                                throw new Exception(
929                                        "JNLPUtilities.canonicalizeJarURL(new URL(\""
930                                                + spec + "\")) returned null.");
931                            }
932                        }
933                    }
934
935                    // Verify that it can be opened
936                    InputStream urlStream = specURL.openStream();
937                    urlStream.close();
938                    return specURL;
939                } catch (Exception ex3) {
940                    // Use a very verbose message in case opening
941                    // the configuration fails under Web Start.
942                    // Without this error message, users will
943                    // have no hope of telling us why Web Start failed.
944                    IOException exception = new IOException("File not found: '"
945                            + spec + "'\n caused by:\n" + ex + "\n AND:\n"
946                            + throwable + "\n AND:\n" + ex3);
947
948                    // IOException does not have a cause argument
949                    exception.initCause(ex3);
950                    throw exception;
951                }
952            }
953        }
954    }

1470    /** Parse the command-line arguments.
1471     *  @param args The command-line arguments to be parsed.
1472     *  @exception Exception If an argument is not understood or triggers
1473     *   an error.
1474     */
1475    protected void _parseArgs(String[] args) throws Exception {
1476        if (args.length > 0) {
1477            _configuration = _createDefaultConfiguration();
1478        } else {
1479            _configuration = _createEmptyConfiguration();
1480        }
1481
1482        for (int i = 0; i < args.length; i++) {
1483            String arg = args[i];
1484
1485            if (_parseArg(arg) == false) {
1486                if (arg.trim().startsWith("-")) {
1487                    if (i >= args.length - 1) {
1488                        throw new IllegalActionException(
1489                                "Cannot set " + "parameter " + arg
1490                                        + " when no value is " + "given.");
1491                    }
1492
1493                    // Save in case this is a parameter name and value.
1494                    _parameterNames.add(arg.substring(1));
1495                    _parameterValues.add(args[i + 1]);
1496                    i++;
1497                } else {
1498                    // Unrecognized option.
1499                    throw new IllegalActionException(
1500                            "Unrecognized option: " + arg);
1501                }
1502            }
1503        }
1504
1505        if (_expectingClass) {
1506            throw new IllegalActionException("Missing classname.");
1507        }
1508
1509        // Check saved options to see whether any is setting an attribute.
1510        Iterator names = _parameterNames.iterator();
1511        Iterator values = _parameterValues.iterator();
1512
1513        while (names.hasNext() && values.hasNext()) {
1514            String name = (String) names.next();
1515            String value = (String) values.next();
1516
1517            boolean match = false;
1518            ModelDirectory directory = _configuration.getDirectory();
1519
1520            if (directory == null) {
1521                throw new InternalErrorException("No model directory!");
1522            }
1523
1524            Iterator effigies = directory.entityList(Effigy.class).iterator();
1525
1526            while (effigies.hasNext()) {
1527                Effigy effigy = (Effigy) effigies.next();
1528
1529                if (effigy instanceof PtolemyEffigy) {
1530                    NamedObj model = ((PtolemyEffigy) effigy).getModel();
1531
1532                    // System.out.println("model = " + model.getFullName());
1533                    Attribute attribute = model.getAttribute(name);
1534
1535                    if (attribute instanceof Settable) {
1536                        match = true;
1537
1538                        // Use a MoMLChangeRequest so that visual rendition (if
1539                        // any) is updated and listeners are notified.
1540                        String moml = "<property name=\"" + name + "\" value=\""
1541                                + value + "\"/>";
1542                        MoMLChangeRequest request = new MoMLChangeRequest(this,
1543                                model, moml);
1544                        model.requestChange(request);
1545
1546                        /* Formerly (before the change request):
1547                         ((Settable)attribute).setExpression(value);
1548                         if (attribute instanceof Variable) {
1549                         // Force evaluation so that listeners are notified.
1550                         ((Variable)attribute).getToken();
1551                         }
1552                         */
1553                    }
1554
1555                    if (model instanceof CompositeActor) {
1556                        Director director = ((CompositeActor) model)
1557                                .getDirector();
1558
1559                        if (director != null) {
1560                            attribute = director.getAttribute(name);
1561
1562                            if (attribute instanceof Settable) {
1563                                match = true;
1564
1565                                // Use a MoMLChangeRequest so that visual rendition (if
1566                                // any) is updated and listeners are notified.
1567                                String moml = "<property name=\"" + name
1568                                        + "\" value=\"" + value + "\"/>";
1569                                MoMLChangeRequest request = new MoMLChangeRequest(
1570                                        this, director, moml);
1571                                director.requestChange(request);
1572
1573                                /* Formerly (before change request):
1574                                 ((Settable)attribute).setExpression(value);
1575                                 if (attribute instanceof Variable) {
1576                                 // Force evaluation so that listeners
1577                                 // are notified.
1578                                 ((Variable)attribute).getToken();
1579                                 }
1580                                 */
1581                            }
1582                        }
1583                    }
1584                }
1585            }
1586

1588                // Unrecognized option.
1589                throw new IllegalActionException("Unrecognized option: "
1590                        + "No parameter exists with name " + name);
1591            }
